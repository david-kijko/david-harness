# Forensic Ingest v2

Ingest any codebase into a structured NotebookLM notebook. Each source = one logical directory group, packaged by repomix. Agents query with selective source names for precision.

## Core Principle

**Group first, then repomix.** Not the other way around.

The pipeline analyzes directory structure to create logical groups, then runs repomix per group. Each repomix output is a self-contained NLM source with accurate file listings. No post-hoc splitting, no inflated char counts.

## Pipeline

```
plan groups (by directory) -> repomix per group -> upload to NLM -> code map index -> verify
```

## Execution

### Option A: Full Pipeline (Recommended)

```bash
cd /home/david/Projects/Kijko_Docs
python -m forensic_ingest.ingest /path/to/repo --name my-project
```

This runs all 5 steps and produces:
- NotebookLM notebook with one source per logical group
- Registry JSON at `/tmp/forensic-ingest/<name>/<name>-registry.json`
- Repomix outputs at `/tmp/forensic-ingest/<name>/sources/*.md`
- Verification report at `/tmp/forensic-ingest/<name>/verification.json`

### Option B: Local Only (No NotebookLM)

```bash
python -m forensic_ingest.ingest /path/to/repo --step local --name my-project
```

Runs steps 1-2 only (plan + repomix). Includes verification audit.

### Option C: Dry-Run (Plan Only)

```bash
python -m forensic_ingest.ingest /path/to/repo --step plan --name my-project
```

Shows planned groups without running repomix or uploading. Inspect the plan at `/tmp/forensic-ingest/<name>/plan.json`.

### Option D: Step-by-Step

```bash
python -m forensic_ingest.ingest /path/to/repo --step plan         # 1. Plan groups
python -m forensic_ingest.ingest /path/to/repo --step repomix      # 2. Run repomix per group
python -m forensic_ingest.ingest /path/to/repo --step upload -n X  # 3. Upload to NLM
python -m forensic_ingest.ingest /path/to/repo --step index -n X   # 4. Build index
python -m forensic_ingest.ingest /path/to/repo --step verify       # 5. Verify
```

### Option E: MCP-Direct (Agent Without CLI)

If running as an agent with NotebookLM MCP tools but no CLI:

1. Use Bash to run `python -m forensic_ingest.ingest /path/to/repo --step local --name X`
2. Read generated sources from `/tmp/forensic-ingest/X/sources/*.md`
3. Upload each source via MCP: `mcp__notebooklm__notebook_add_text(notebook_id, text, title)`
4. Upload CODE-MAP-INDEX from `/tmp/forensic-ingest/X/CODE-MAP-INDEX.md`

## Key Options

| Flag | Default | Purpose |
|------|---------|---------|
| `--max-sources N` | 80 | Max NLM sources per notebook. Reduce for large repos. |
| `--max-chars N` | 90000 | Max chars per source (95K NLM limit minus headroom) |
| `--no-tests` | off | Exclude test files from ingestion |
| `--name NAME` | dir name | Project name for the notebook |

## How Grouping Works

1. **List files** via `git ls-files` (respects .gitignore) or os.walk fallback
2. **Group by 2nd-level directory** (e.g., `src/module/` = one group)
3. **Split oversized groups** by going deeper into subdirectories
4. **Merge tiny groups** (<1K chars) into nearest neighbor
5. **Budget cap**: if over `--max-sources`, keep merging smallest pairs

For small repos (<90K total): everything in one source.
For large repos (1000+ files): groups are directory-based, sized to stay under `--max-chars`.

## Source Format

Each source is a **raw repomix markdown output** — self-contained with:
- File structure listing (generated by repomix)
- Full source code for every file in the group
- Accurate file paths preserved from repo root

No manual formatting, no inflated char counts. What repomix outputs = what NLM gets.

## CODE-MAP-INDEX

The index reports honestly:

| Field | Meaning |
|-------|---------|
| Raw Chars | Sum of actual file sizes in the group |
| Source Chars | Repomix output size (includes markdown formatting) |
| Overhead | Percentage difference (typically 15-40%) |
| File Manifests | Exact list of files in each source |

## Verification

Step 5 compares:
- Files in repo vs files in plan (catches missed files)
- Files in plan vs files in repomix output (catches repomix filter issues)
- Raw chars vs source chars (reports honest overhead)

Output: `verification.json` with `verified: true/false` and lists of any gaps.

## Prerequisites

All pre-installed:
- `repomix` — codebase dump per group (`--include` patterns, `--style markdown`)
- `notebooklm-mcp` — NotebookLM API (pipx, auth via `notebooklm-mcp-auth`)

## Troubleshooting

| Issue | Fix |
|-------|-----|
| NotebookLM auth error | Run `notebooklm-mcp-auth` |
| repomix not found | `npm install -g repomix` |
| Too many sources | Use `--max-sources 40` or lower |
| Source too large | Use `--max-chars 50000` to force smaller groups |
| Verification shows missing files | Check .gitignore or repomix --include patterns |
| Files in wrong source | Adjust directory structure or use `--max-chars` to force splits |
